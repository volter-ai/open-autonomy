# R9 closure

R9 closes only when migration is total, dual compilation is independently replayed, cutover is reversible and authenticated, and the canonical repository profile has executed through the observed v2 controller path.

| Obligation | Constructive evidence |
|---|---|
| R9-SEM-1 | Every discovered v1 construct has exactly one resolving disposition; the native Organization IR is validated, solved, fixed-point lowered, and exactly round-tripped through the explicitly retained v1 runtime ABI. |
| R9-REF-1 | Every repository profile is compiled twice and independently replayed. Compiler source locks, invocation certificates, raw outputs, generated file bytes, installed command dispatchers, owned state, and routing/review effects are bound into the differential report. |
| R9-EVO-1 | Signed evidence gates shadow, fractional canary, cutover, and rollback. Checkpoints replay under external trust; post-cutover mutations translate on rollback; removal uses a closed nine-criterion decision and an irreversible tombstone. |
| R9-DOG-1 | `r9-repository-dogfood.json` is reproducibly generated from `profiles/self-driving/ir.yml`; it records the actual v2 compiler identity, certificate, emitted artifacts, signed shadow/canary/cutover chain, active v2 checkpoint, and canonical owned-state effect. |

Verification requires the focused R9 tests, generated-evidence freshness check, typecheck, full core gate, and a final skeptical review with zero unresolved blockers.
