# Organization universality punchlist — pre-build adversarial review

Review date: 2026-07-16. Reviewed revision: `organization-universality-punchlist.v2`.

## Initial verdict

The v1 U0–U18 punchlist was not build-ready. It could close with arbitrarily low coverage, count opaque wrappers as
representation, weaken observations until a target passed, populate every matrix cell with incompatibility, and let
frontends omit difficult native facts from inventories they authored themselves. It also selected target backends
after seeing source encodings, permitted holdout-driven language extensions to improve the holdout score, conflated
static certificates with native execution, and coupled semantic reproduction to product release.

## Required corrections and disposition

| Finding | v2 disposition |
|---|---|
| No minimum universality result | U0 now sets non-lowerable numeric campaign floors; U20 fails honestly below them |
| Opaque/extension escape hatch | U5 credit lattice excludes opaque, local extension, abstraction, unsupported, and inexpressible facts from canonical credit |
| Weak observation profile | U3 source-mandatory observations cannot be weakened by optional profiles |
| Universal rejection completes matrix | U17 separates 100% accounting from compatibility, compilation, portability, and execution numerators |
| Frontend self-certifies fact universe | U4 inventories are independently frozen before U6 frontends |
| Backend selection after results | U2 freezes component and composition populations before backend work |
| Holdout repaired after inspection | U8 retains frozen-core score; extensions require a new campaign/version |
| Static evidence presented as execution | U18 stratifies static, conformance, native-run, and independent assurance |
| Unbounded prompt equivalence | U14 scopes certificates to fixed harness/model/context/probe observations |
| “Minimal core” overclaim | U12 separately verifies minimal, bounded-irreducible, and explanatory cores |
| Weak independent reproduction | U21 reproduces inventory through final estimator and requires second frontend and backend implementations |
| Semantic and release claims coupled | U20/U21 establish semantic result; U22 separately establishes public operability |

## Second-pass verdict

PASS to begin U0 only. The revised U0–U22 graph freezes claim rules before evidence, separates representation from
interoperation and incompatibility, and makes a failed universality campaign a valid honest outcome rather than
allowing the claim to shrink. This verdict does not pre-approve later checkpoints: every checkpoint still requires its
own falsification evidence and skeptical review. U0 must instantiate exact formulas above the normative floors before
U1 census work begins.
