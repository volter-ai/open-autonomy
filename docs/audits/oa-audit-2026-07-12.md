**Verdict:** FAIL

**Failure:** Three different strings, none matching. The `simple-gh-sdlc` and `simple-sdlc` presets are substantively different — `simple-gh-sdlc` requires PR coupling ("review on a PR, merged PR for done"), while `simple-sdlc` is PR-free ("PR-FREE by design: it never requires a pull request"). The installed validation grammar (`simple-sdlc`) does not enforce PR coupling, but the declared preset (`simple-gh-sdlc`) claims it does. An agent enforcing the wrong preset silently validates against the wrong grammar.

This is exactly the class of defect the check is designed to catch: a three-name mismatch between the declared preset, the installed validation, and the compiling profile.
